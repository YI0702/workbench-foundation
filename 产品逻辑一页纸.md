## ① 定位 · 基座做什么 / 不做什么

**基座 = 把五个模块串成"一个产品"的形态骨架，不是又一个独立模块。**

| ✅ 基座做 | ❌ 基座不做 |
|---------|----------|
| 导航与聚合（首页是入口） | 不是"第六个模块" |
| 共用能力的载体 | 不做模块内工作流和具体视觉 |
| 用户体验上的统一性 | 不是 PM 一个人的看板 |
| 跨模块流转的协议 | 不是完成态——跟模块同步迭代 |

---

## ② PM 的统筹原则 · 最小化干预

> **PM 只在 1 件事上守底线：入口路径与跳转契约**

其余（视觉一致 / 数据口径 / Skills 提报 / 看板卡片 / 关键词路由）都是讨论共识可迭代。**底盘稳，模块才能跑得快。我提的不是约束，是底盘。**

---

## ③ 基座的双视图 · 形态层 × 能力层

**A · 用户视角（基座四层）**
1. **AI 对话入口**·语义路由 → 知道要做什么的人
2. **场景化 Prompt 卡片**·6 张高频任务 → 不知道怎么开始的人
3. **自定义视角看板**·赛道/场景/预算/SDK → 业务 owner
4. **Skills 资产晾晒区**·核心强定义+分支个人贡献 → 所有人

↓ 调用

**B · 技术视角（共用基础设施）**
- AI 对话路由 · 通用组件库 · 统一鉴权
- 数据口径共识 · 操作日志 · Skills 嵌入框架

> 两者是同一件事的两面——基座是表层，共用基础设施是底层。

## ④ PM × 模块的协同接口

| 维度 | 性质 | 谁拍板 |
|------|------|-------|
| AI 路由关键词 | 🟡 共建对齐 | owner 提报 → PM 检查冲突 |
| 首页看板卡片 | 🟡 共建对齐 | owner 设计 → PM 走查 |
| Skills 提报上架 | 🟡 共建对齐 | 分层审核（见第⑤节） |
| 数据口径 | 🟡 共建对齐 | 业务理解共识 → 指标 owner 自定义 |
| 视觉一致性 | 🟡 共建对齐 | 通用组件库 + 设计走查（**路演后启动**） |
| 入口路径与跳转契约 | 🔴 PM 兜底 | 跳转参数协议 PM 定 |

> 三种执行模式：🟢模块自定义 / 🟡共建对齐 / 🔴PM兜底。模块内部（工作流、界面、技术）属于 🟢，本表不涉及。

**"入口路径与跳转契约"具体指什么**：用户从基座跳到模块时 URL 参数和上下文怎么传。例：AI 对话框输入"金融行业模板效果" → 跳转洞察归因时 URL 是 `?industry=金融&module=洞察归因`，参数命名必须统一。

## ⑤ Skills 管理 · 核心强定义 + 分支共创

> **导向**：工作台对核心使用流程做强定义约束 · Skills 共创满足分支流程的个人贡献。**Skills 是主流程的补充和增强，不是主流程的替代**。

**分层审核（讨论稿）**

| 类型 | 判断 | 审核路径 |
|------|------|---------|
| **A 类** | 涉线上配置 / 跨模块 / 用户隐私 | PM + 模块 owner + 安全合规 |
| **B 类** | 仅影响某模块查询/展示 | 对应模块 owner 审核 |
| **C 类** | 个人自用 | 不进资产库，自己空间 |

⚠️ **关键讨论点**：Skill 不只是上架可用——还要和当前模块工作流融合（触发位置、输入输出、失败兜底、版本管理）。本节先给导向，融合细节后续立项讨论。

## ⑥ 用户主路径 · 4 种使用方式（Demo 里都能点）

| # | 类型 | 流程 | 支撑 |
|---|-----|------|------|
| 01 | 知道要做什么 | AI 对话框输入意图 → 关键词识别 → 路由到模块 → prompt 自动填充 | 第 1 层 AI 对话 |
| 02 | 不确定怎么开始 | 浏览场景卡片（6 张）→ 点击高频任务 → 自动带 prompt 进模块 | 第 2 层 场景卡片 |
| 03 | 业务 owner 看大盘 | 视角切换 → 跨模块聚合数据 → 深入对应模块 | 第 3 层 视角看板 |
| 04 | 复用团队资产 | 浏览 Skills 资产 → 点击触发 → 经关键词路由进模块 → 模块内承载执行 | 第 4 层 Skills 共创 |

## ⑦ 风险识别 · 语义理解兜底

> 参考 Anthropic / Cursor / Microsoft Copilot Studio（confidence 70% / fallback rate 阈值 20%）

| 失败模式 | 兜底路径 |
|---------|---------|
| 意图无法识别 | **场景卡片兜底**·引导用 6 张高频任务卡片 |
| 置信度不足（多意图） | **意图澄清**·反问"是 A 还是 B" |
| 重复误解 | **Prompt 模板**·"我要 XX → 推荐 YY" |
| 超出范围 | **边界声明**·"工作台不处理 XX，建议 YY" |

## ⑧ 跨职能协同（建议稿 · 待与对应角色 1on1 确认）
